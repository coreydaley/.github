# Support

## Where to get help

- **Bug reports & feature requests:** Open an issue in the relevant repository using the provided templates.
- **Questions about a specific project:** Start a [GitHub Discussion](https://github.com/coreydaley) in that repository if discussions are enabled, or open an issue tagged `question`.
- **General inquiries:** Email [corey@coreydaley.dev](mailto:corey@coreydaley.dev) or reach out via [LinkedIn](https://linkedin.com/in/coreydaley-dev).
- **Blog & articles:** Visit [coreydaley.dev](https://coreydaley.dev) for write-ups on AI, automation, and developer tooling.

## Before opening an issue

1. Check the repository's `README.md` and any linked documentation.
2. Search existing open and closed issues — your question may already be answered.
3. Make sure you're on the latest version.

## Response times

These are open-source projects maintained in my spare time. I'll do my best to respond promptly, but cannot guarantee SLAs. Security issues are prioritized — see [SECURITY.md](SECURITY.md).
